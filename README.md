# KNOCK Onboarding FE (Flutter)

이 문서는 온보딩 과제(게시글/댓글) 구현 및 테스트/품질 기준을 충족하기 위한 README입니다.

## 0. TL;DR
- 앱 실행: `flutter run`
- 테스트: `flutter test`
- (커버리지) `flutter test --coverage`

## 1. 프로젝트 개요
- **앱 이름**: Knock
- **기능**: 게시글/댓글 CRUD, 좋아요(낙관적 업데이트), 이미지 업로드(base64), 무한 스크롤 댓글, 검색/필터
- **상태관리**: BLoC (`flutter_bloc`)
- **로컬 캐시**: `shared_preferences`
- **네트워크**: `dio` + json-server

## 2. 실행 방법
### 2.1 Flutter
1) 의존성 설치
- `flutter pub get`
2) 실행
- `flutter run`

### 2.2 json-server
- 로컬에서 json-server를 띄워 `/posts`, `/replies` API를 제공합니다.
- 데이터는 `db.json`을 사용합니다.

## 3. 화면(페이지)
- `PostListPage`: 게시글 목록, 검색(디바운스 300ms), 필터 BottomSheet
- `PostDetailPage`: 게시글 상세, 댓글(10개 단위 로드), 댓글 총 개수 표시
- `PostCreatePage`: 게시글 작성/수정, 이미지 선택/미리보기
- (Week4) `PostFilterSheet`: 필터 BottomSheet(유형/작성자/기간)

## 4. 아키텍처
### 4.1 Feature 구조
- `features/post/domain`: Entity/Repository 인터페이스
- `features/post/data`: DataSource(Remote/Local), Model, Repository 구현
- `features/post/presentation`: Page/UI, BLoC

### 4.2 Repository 정책
- Remote 우선 → 실패 시 Local fallback
- Local은 shared_preferences 기반 캐시

## 5. 기능 상세
### 5.1 검색 (Debounce 300ms)
- `PostListPage`에서 입력 변화에 대해 300ms 디바운스를 적용합니다.
- 목표 정확도: 300ms ± 50ms

### 5.2 필터 (BottomSheet)
- 동시에 3가지 필터를 조합할 수 있습니다.
  - 유형(type): 전체/게시글/질문
  - 작성자(author)
  - 기간(dateFrom/dateTo)

### 5.3 댓글 페이지네이션
- 댓글은 스크롤 시 10개 단위로 추가 로드합니다.
- 상세 진입 시에는 댓글 총 개수를 표시합니다.

### 5.4 작성일시 표시
- 게시글/댓글의 작성일시를 `yyyy.MM.dd HH:mm` 포맷으로 표시합니다.

### 5.5 성능 최적화
- 목록에서 `reversed.toList()`로 매 리빌드마다 새 리스트를 만들지 않도록 제거했습니다.
- 불필요한 객체 생성을 줄이기 위해 가능한 부분에 `const`를 추가했습니다.

## 6. 테스트
### 6.1 실행
- `flutter test`

### 6.2 커버리지
- `flutter test --coverage`
- 결과 파일: `coverage/lcov.info`

### 6.3 테스트 구성
- Entity 테스트
- Repository 테스트 (remote/local 모킹)
- BLoC 테스트 (bloc_test + mocktail)

## 7. 트러블슈팅
- mocktail `any()` 사용 시 `registerFallbackValue`를 반드시 등록해야 합니다.
- json-server 쿼리의 AND/OR 특성 때문에 검색은 클라이언트 필터를 병행합니다.
